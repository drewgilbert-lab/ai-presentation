# Campaign Philosophy: Problems, Not Use Cases

<div class="text-[15px] text-hg-dark mt-4">Campaigns are now organized around a persona's <span class="font-bold text-hg-navy">problem</span>, which may include one or more use cases.</div>

<div class="grid grid-cols-2 gap-6 mt-6">
  <div class="hg-card">
    <div class="hg-card-header">Why it matters</div>
    <div class="hg-card-body">
      <ul class="text-[13px]">
        <li>People think in problems, not use cases</li>
        <li>One problem may map to multiple HG use cases</li>
        <li>A brief may reference multiple use cases on purpose, not by mistake</li>
        <li>The core solution still anchors to a primary use case</li>
      </ul>
    </div>
  </div>
  <div class="hg-card">
    <div class="hg-card-header">In practice</div>
    <div class="hg-card-body">
      <div class="text-[13px]">When a campaign is named after a use case, the assets can speak to the problems that persona faces.</div>
      <div class="text-[13px] mt-3">Those problems may map to a different use case or span several use cases. Content can address adjacent solutions.</div>
    </div>
  </div>
</div>

# Step 6: Why HTML, and Load Content First

<div class="grid grid-cols-2 gap-8 mt-6">
  <div class="hg-card">
    <div class="hg-card-header">Why HTML</div>
    <div class="hg-card-body text-[13px]">The direct PDF conversion tool (which replaced Playwright) handles design and spacing poorly; it cuts off mid-sentence with no content grouping. The HTML approach understands logical structure and handles page breaks intelligently. Example: a section that would start in the lower 25% of a page gets pushed to the next page.</div>
  </div>
  <div class="hg-card">
    <div class="hg-card-header">Load content first</div>
    <div class="hg-card-body text-[13px]">Make sure the Google Doc content is loaded into the chat in the very first message, or the skill may produce unexpected results. The skill is designed not to rewrite content; it takes the approved content as-is and applies the design template.</div>
